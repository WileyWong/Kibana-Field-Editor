# Kibana Dev Tools 字段值修改插件 — 需求文档

## 1. 概述

### 1.1 产品名称
Kibana Field Editor（Chrome Extension）

### 1.2 目标
为 Kibana Dev Tools 页面提供一个 Chrome 插件，允许用户在查看 `_search` 结果时，直接点击 `_source` 中的字段，快速修改字段值并通过 ES `_update` 接口持久化更新。

### 1.3 适用范围
- Chrome 浏览器（MV3 扩展）
- **首要目标环境：Kibana 7.5.1**（Dev Tools Console 基于 Ace Editor，URL 为 `/app/kibana#/dev_tools/console`）
- 通配所有域名：`*/app/kibana*`（7.x 旧版 URL 形态）+ `*/app/dev_tools*`（8.x 新版 URL 形态，向后兼容预留）
- **编辑器兼容**：Kibana Dev Tools Console 在 **8.16** 从 Ace 迁移到 Monaco（[Elastic 官方博客](https://www.elastic.co/blog/dev-tools-console-kibana)）。本插件**两种编辑器均已适配，全版本可用**：
  - **< 8.16（全部 7.x、8.0~8.15）= Ace**：完整支持（复用 `window.ace`，可就地回写结果区）
  - **8.16+（含 8.17.x）= Monaco**：已适配（拦截 `_search` 响应 + DOM 点击定位）；**差异**：更新成功后结果区不就地回写，需重新执行查询查看最新值

## 3. 功能需求

### 3.1 页面检测与激活

| 编号 | 需求 |
|------|------|
| FR-01 | 插件仅在 Dev Tools Console 页激活：`matches` 用 `*/app/kibana*`+`*/app/dev_tools*`，加载后运行时校验 `location.hash` 含 `dev_tools/console`，并监听 `hashchange` 处理 SPA 路由切换 |
| FR-01a | **编辑器类型探测**（持续观察 DOM，直到命中或超时）：<br>- 检测到 `.conApp__output` → **Ace 版**，注入 `injected.js` 走 Ace 链路<br>- 检测到 `[data-test-subj="consoleMonacoOutput"]` 或 `.monaco-editor` → **Monaco 版**，注入 `injected-monaco.js` 走 Monaco 链路<br>- 兜底超时仍无任何已知编辑器 → 仅 console `debug` 提示，完全静默（不注入/不绑定，等同插件不存在） |
| FR-02 | 激活后（仅支持环境下），插件异步调用 `GET /`（通过 Kibana proxy）获取 ES 版本信息，存入内存备用 |

### 3.2 点击检测与浮层展示

| 编号 | 需求 |
|------|------|
| FR-03 | 监听 Kibana Dev Tools **结果区** Ace Editor 的鼠标点击事件 |
| FR-04 | 点击后，解析结果区 JSON 全文，判断点击位置是否落在某条 hit 的 `_source` 下的**可编辑字段**上 |
| FR-05 | **可编辑字段**定义：<br>① 顶层标量字段（string/number/boolean/null）→ doc 更新<br>② 嵌套对象内的标量字段（不含数组）→ doc 更新<br>③ 数组内的**直接标量元素**（如 `tags[0]`）→ 触发整个数组编辑<br>④ **数组内对象的标量字段**（如 `WorkExperienceList[0].StartDate`）→ painless script 精准更新该字段 |
| FR-06 | **不触发**的情况：<br>① 点击位置不在 `_source` 内<br>② 点击在对象或数组的 key 名上（如 `"tags":`、`"user":`）<br>③ 点击在值为**对象**的字段上（无论点 key 还是 value，MVP 不支持对象整体编辑）<br>④ 点击在值为**数组**的 key 名上（数组仅在点击其内部元素值时触发，见 FR-05③）<br>⑤ 点击在 `_id`、`_index`、`_score` 等元数据字段上 |
| FR-07 | 命中可编辑字段时，在**该字段所在行的最右侧**显示一个半透明浮层按钮，文案为"修改值" |
| FR-08 | 同一时间最多显示一个浮层；用户点击页面其他区域时，浮层自动消失 |

### 3.3 修改弹窗

| 编号 | 需求 |
|------|------|
| FR-09 | 用户点击"修改值"浮层按钮后，弹出**模态弹窗（带遮罩层）**；弹窗打开期间，结果区及页面其他区域不可交互，以避免出现多个弹窗/浮层的状态冲突 |
| FR-10 | 弹窗标题："修改字段值" |
| FR-11 | 弹窗内容区：<br>- 字段路径（只读），如 `hits.hits[0]._source.user.name`<br>- 文本输入区（textarea），默认显示原始值：<br>&nbsp;&nbsp;• 标量字段：直接显示原始值文本<br>&nbsp;&nbsp;• 数组字段：显示整个数组的 JSON 文本（格式化） |
| FR-12 | "确认"按钮：默认禁用，仅当输入值与原始值**不同**时启用 |
| FR-13 | "取消"按钮 / 右上角 × 号：关闭弹窗，无副作用 |

### 3.4 值类型转换

| 编号 | 需求 |
|------|------|
| FR-14 | 提交前，根据**原值类型**对用户输入做类型转换：<br>- 原值 number → `Number(input)`，若为 NaN 则提示"请输入合法数字"并阻止提交<br>- 原值 boolean → 严格匹配 `true`/`false`（不区分大小写），否则提示"请输入 true 或 false"<br>- 原值 string → 直接使用，无需转换<br>- 原值 null → 智能推断：`null`→null、`true`/`false`→boolean、纯数字→number、其他→string；弹窗显示提示"原值为 null，将根据输入自动推断类型"<br>- 数组 → `JSON.parse(input)`，解析失败提示"JSON 格式错误" |

### 3.5 调用 ES 更新

| 编号 | 需求 |
|------|------|
| FR-15 | 通过 Kibana proxy 接口 `POST /api/console/proxy` 调用 ES `_update` API；`path` 为 `/{_index}/_update/{_id}`，`_index`/`_id` 取自该条 hit，Headers 带 `kbn-xsrf: true` |
| FR-16 | **更新模式按字段所处位置自动选择**：<br>- **不在数组内的标量/顶层数组**（字段路径不含数组下标）→ **doc 部分更新**：`{ "doc": { "<dot_path>": <new_value> } }`<br>- **数组内对象的字段**（路径含数组下标，如 `WorkExperienceList[0].StartDate`）→ **painless script 更新**：`{ "script": { "lang":"painless", "source": "<painlessPath> = params.v", "params": { "v": <new_value> } } }`，其中 `<painlessPath>` 形如 `ctx._source['WorkExperienceList'][0]['StartDate']` |
| FR-17 | 嵌套字段（不含数组）使用点号路径，如 `user.name`，走 doc 更新（ES `_update` 支持点号路径直接更新嵌套字段）。<br>选择 script 模式的原因：ES `_update` 的 doc 合并**无法定位数组内某个元素的字段**，点号路径 `list.field` 对数组无意义（会导致 `result:"noop"`），故数组内字段必须用 painless 按下标精准赋值 |

### 3.6 结果展示与回显

| 编号 | 需求 |
|------|------|
| FR-18 | 调用期间，弹窗内显示 loading 状态，"确认"按钮禁用 |
| FR-19 | **更新成功**：<br>① 弹窗内展示 ES 返回的完整响应 JSON<br>② 同时，将 Kibana 结果区 Ace Editor 中该字段的值**局部替换**为新值（使用 `session.replace()` 精准替换原值 range；结果区只读，替换前临时解除只读，替换后恢复）<br>③ 用户手动关闭弹窗 |
| FR-20 | **更新失败**：<br>① 弹窗内展示 ES 返回的错误响应 JSON<br>② **不修改** Ace Editor 中的任何内容<br>③ 用户手动关闭弹窗 |
| FR-21 | 弹窗关闭后，用户可继续点击其他字段进行修改（不要求刷新结果）。由于弹窗为模态（FR-09），同一时间只存在一个浮层与一个弹窗，不存在并发交互 |

## 4. 非功能需求

| 编号 | 需求 |
|------|------|
| NFR-01 | 性能：点击到浮层出现 < 100ms（万行 JSON 内） |
| NFR-02 | 兼容：**全版本支持**。Ace 版（< 8.16，实测 7.5.1）与 Monaco 版（8.16+，实测 8.17.3）通过独立适配层（`injected.js` / `injected-monaco.js`）分别实现，上层 locate/convert/proxy/UI 逻辑完全复用 |
| NFR-03 | 隔离：使用 Shadow DOM 隔离插件 UI 样式，避免与 Kibana 样式互相污染 |
| NFR-04 | 安全：不存储任何凭据，完全复用浏览器已有的 Kibana session |
| NFR-05 | 无侵入：不修改 Kibana 页面原有功能逻辑，仅附加 UI 和事件监听 |

---

## 5. 不做（MVP 排除项）

| 排除项 | 说明 |
|--------|------|
| 操作日志/回滚 | 不提供撤销功能 |
| 批量修改 | 仅支持单字段/单数组逐条修改 |
| 对象整体编辑 | 点击对象 key 名不触发编辑 |
| 权限管理 | 复用 Kibana 原有权限，无额外鉴权 |

## 6. 交互流程图（文字版）

```
用户在 Kibana Dev Tools 执行 _search
        │
        ▼
结果区展示 JSON（Ace Editor）
        │
        ▼
用户点击 _source 中的字段名或值
        │
        ▼
 [解析点击位置是否在可编辑字段上]
        │
   Yes ─┤── No → 无反应
        │
        ▼
该行最右侧出现"修改值"浮层
        │
用户点击 ──┤── 点击其他位置 → 浮层消失
        │
        ▼
弹出"修改字段值"弹窗（含原始值）
        │
        ▼
用户修改值 → "确认"按钮启用
        │
        ▼
点击"确认" → 类型校验
        │
   Pass ─┤── Fail → 弹窗内提示错误，不提交
        │
        ▼
调用 Kibana Proxy → ES _update
        │
   成功 ─┤── 失败
   │         │
   ▼         ▼
弹窗展示成功响应    弹窗展示错误响应
Ace 替换新值       Ace 不变
   │         │
   ▼         ▼
用户手动关闭弹窗，可继续操作
```
