# Kibana Dev Tools 字段值修改插件 — 技术方案文档

> **目标环境（已实测确认）**
> - Kibana **7.5.1**（旧版形态）
> - Dev Tools Console 使用 **Ace Editor**（**非 Monaco**）；Kibana 7.11 起才迁移到 Monaco
> - 页面全局暴露 `window.ace`（`typeof === "object"`，可直接使用）
> - 结果区编辑器 DOM 选择器：**`.conApp__output`**（`readOnly:true`）
> - 请求区编辑器：`#ConAppEditor`（`.conApp__editorContent`，`readOnly:false`）
> - URL 形态：`http(s)://<host>/app/kibana#/dev_tools/console`

## 1. 技术选型

| 项 | 选型 | 说明 |
|----|------|------|
| 扩展规范 | Chrome Extension Manifest V3 | 最新规范，Chrome 强制要求 |
| UI 框架 | 原生 JS + Shadow DOM | 轻量、零依赖、样式隔离 |
| JSON 解析 | `jsonc-parser`（或自实现 AST） | 提供带 offset 的 AST，便于定位点击位置 |
| **编辑器交互** | **复用 Kibana 页面内置 Ace Editor（`window.ace`）** | **不打包 Ace，零体积；通过 `ace.edit(domNode)` 反查已有实例** |
| 网络请求 | `fetch`（同源，复用 cookie） | 通过 Kibana proxy 调用 ES |
| 构建工具 | Vite（或 esbuild） | 打包 content script / 注入脚本 |

### 2.1 为什么需要 MAIN world 注入？
- Content Script 运行在 **isolated world**，无法访问页面的 `window.ace` 全局对象与 Kibana 创建的 editor 实例。
- 因此用 `<script>` 标签（指向 `web_accessible_resources` 中的 `injected.js`）注入一段脚本到 **MAIN world**，由它直接操作 Ace，再通过 `window.postMessage` 与 content script 双向通信。
- 实测 `window.ace` 全局可用，因此 injected 脚本逻辑简单：`window.ace.edit(document.querySelector('.conApp__output'))` 即可拿到结果区实例。

### 3.1 manifest.json

```json
{
  "manifest_version": 3,

  "version": "1.0.0",
  "description": "在 Kibana Dev Tools 中快速修改 _source 字段值",
  "host_permissions": ["*://*/app/kibana*", "*://*/app/dev_tools*"],
  "content_scripts": [

    }
  ],
  "web_accessible_resources": [

      "matches": ["*://*/*"]
    }
  ]

```

> **URL 匹配说明（基于实测 7.5.1）**：本环境的 Dev Tools URL 为 `/app/kibana#/dev_tools/console`，路径是 `/app/kibana`，**真正的子页信息在 hash 里**。因此：
> - `matches` 用 `*://*/app/kibana*`（7.x 旧版）+ `*://*/app/dev_tools*`（7.11+/8.x 新版，向后兼容）。
> - 由于 `matches` 无法匹配 hash，content script 加载后必须**运行时校验 `location.hash` 是否包含 `dev_tools/console`**，并监听 hash 变化（SPA 路由切换），仅在 Console 页激活逻辑。

### 3.2 injected.js（MAIN world）

职责：直连页面内置 Ace Editor，做事件监听与文本替换。

```js
// 伪代码（Ace 版）
(function () {
  const OUTPUT_SELECTOR = '.conApp__output';   // 实测: 结果区 DOM

  function getResultEditor() {
    if (typeof window.ace === 'undefined') return null;
    const node = document.querySelector(OUTPUT_SELECTOR);
    if (!node) return null;
    // ace.edit(已存在的容器) 会返回已绑定的实例, 不会重建
    return window.ace.edit(node);
  }

  function init() {
    const editor = getResultEditor();
    if (!editor) { setTimeout(init, 500); return; } // 轮询直到结果区出现

    // Ace 点击监听
    editor.on('mousedown', (e) => {
      // e 是 Ace MouseEvent, 取点击的文档坐标 {row, column}
      const pos = e.getDocumentPosition();          // { row, column }  (0-based)
      if (!pos) return;
      const session = editor.getSession();
      const fullText = session.getValue();
      // Ace: 文档坐标 → 全文字符偏移
      const offset = session.getDocument().positionToIndex(pos);
      window.postMessage({
        source: 'KFE_INJECTED',
        type: 'CLICK',
        payload: {
          fullText,
          offset,
          pos,
          // 屏幕坐标(放浮层用): 行的像素 Y
          screen: editor.renderer.textToScreenCoordinates(pos.row, 0)
        }
      }, window.location.origin);   // 指定目标 origin, 不用 '*'
    });

    // 接收 content script 的替换指令
    window.addEventListener('message', (ev) => {
      // 安全校验: 必须来自本窗口 + 同源 + 自定义标识
      if (ev.source !== window) return;
      if (ev.origin !== window.location.origin) return;
      const d = ev.data;
      if (d?.source !== 'KFE_CONTENT') return;
      if (d.type === 'REPLACE') {
        const session = editor.getSession();
        const doc = session.getDocument();
        // 字符偏移 → 文档坐标
        const startPos = doc.indexToPosition(d.payload.startOffset);
        const endPos = doc.indexToPosition(d.payload.endOffset);
        const Range = window.ace.require('ace/range').Range;
        const range = new Range(startPos.row, startPos.column, endPos.row, endPos.column);
        // 结果区是只读的, 替换前临时解锁, 替换后恢复
        const wasReadOnly = editor.getReadOnly();
        editor.setReadOnly(false);
        session.replace(range, d.payload.newText);
        editor.setReadOnly(wasReadOnly);
      }
    });
  }
  init();
})();
```

> **Ace API 要点**
> - `ace.edit(domNode)`：传入**已存在**的 Ace 容器节点，返回其已绑定实例（不会重建）。
> - `editor.on('mousedown', e => e.getDocumentPosition())`：拿到点击处的 `{row, column}`（0-based）。
> - `session.getDocument().positionToIndex({row,column})` / `indexToPosition(offset)`：文档坐标 ↔ 全文字符偏移互转，用于对接 JSON AST 的 offset。
> - `session.replace(new Range(...), text)`：局部替换，等价于 Monaco 的 `executeEdits`。
> - `editor.renderer.textToScreenCoordinates(row, col)`：拿行的屏幕像素坐标，用于定位浮层。
> - 结果区 `readOnly:true`，替换前需 `setReadOnly(false)`，替换后恢复（用户看不到中间态）。

### 3.3 content.js（ISOLATED world）

职责：路由/版本检测、注入 injected.js、解析 AST、渲染 UI、调用接口。

#### 3.3.0 启动流程与版本兼容检测（编辑器类型探测）

content.js 加载后的总入口流程：

```
content.js 加载
   │
   ▼
校验 location.hash 是否含 'dev_tools/console'  ──否──► 静默, 监听 hashchange 等待切入
   │ 是
   ▼
detectCompatibility()  (带轮询: 最多 3 次 × 500ms)
   │
   ├─ 检测到 .conApp__output (Ace) ──► [支持] 注入 injected.js + 初始化全部功能
   │
   └─ 超时仍无 .conApp__output ──► [不支持]
            │
            ▼
      console 输出一条提示, 随后完全静默
      (不注入 injected.js / 不绑事件 / 不渲染任何 UI, 等同插件不存在)
```

实现（编辑器类型探测，**判据 A**）：

```js
const SUPPORT_SELECTOR = '.conApp__output';        // Ace 结果区(本插件唯一支持目标)
const MONACO_SELECTOR  = '.monaco-editor';         // 仅用于在提示里区分原因

function detectCompatibility({ retries = 3, interval = 500 } = {}) {
  return new Promise((resolve) => {
    let left = retries;
    const tick = () => {
      if (document.querySelector(SUPPORT_SELECTOR)) {
        return resolve({ ok: true, editor: 'ace' });
      }
      if (--left <= 0) {
        const editor = document.querySelector(MONACO_SELECTOR) ? 'monaco' : 'unknown';
        const version = document.querySelector('meta[name="kbn-version"]')?.content || '未知';
        return resolve({ ok: false, editor, version });
      }
      setTimeout(tick, interval);
    };
    tick();
  });
}

async function bootstrap() {
  if (!/dev_tools\/console/.test(location.hash)) return;   // 非 Console 页, 静默
  const compat = await detectCompatibility();
  if (!compat.ok) {
    // 不支持: 仅 console 提示, 然后什么都不做
    console.info(
      '[Kibana Field Editor] 当前环境不受支持，插件未激活。' +
      `（检测到编辑器: ${compat.editor}；Kibana 版本: ${compat.version}）` +
      ' 本插件当前仅支持基于 Ace Editor 的 Kibana（约 7.5.x，<7.11）。'
    );
    return;   // 完全静默, 等同插件不存在
  }
  // 支持: 正式初始化
  injectMainWorldScript();
  initFeature();          // 绑定 message 监听、版本探测(FR-02)、UI 等
}

// SPA 路由: 用户从别的子页切到 Console 时再尝试一次
window.addEventListener('hashchange', () => { bootstrap(); });
bootstrap();
```

> 说明：
> - **判据只看编辑器类型**（有没有 `.conApp__output`），不依赖版本号；版本号仅用于 console 提示文案，便于用户反馈。
> - 不支持时**不注入 `injected.js`、不绑定任何事件、不渲染任何 UI**，零副作用。
> - `bootstrap()` 用一个幂等标志位防止 `hashchange` 重复初始化（实现略）。

#### 3.3.1 注入 injected.js（仅支持环境下执行）
```js
function injectMainWorldScript() {
  const s = document.createElement('script');
  s.src = chrome.runtime.getURL('injected.js');
  s.onload = () => s.remove();
  (document.head || document.documentElement).appendChild(s);
}
```

**postMessage 通信安全（两端统一约定）**：injected.js ↔ content.js 通过 `window.postMessage` 通信，由于同页面其他脚本也能监听 message，需双向加固：

```js
// content.js 接收 injected 的 CLICK 消息
window.addEventListener('message', (ev) => {
  if (ev.source !== window) return;                       // 必须来自本窗口
  if (ev.origin !== window.location.origin) return;       // 必须同源
  const d = ev.data;
  if (d?.source !== 'KFE_INJECTED') return;               // 必须带自定义标识
  if (d.type === 'CLICK') handleClick(d.payload);
});

// content.js 发送 REPLACE 给 injected (target origin 指定为本源)
function postReplaceToAce(range, newText) {
  window.postMessage(
    { source: 'KFE_CONTENT', type: 'REPLACE',
      payload: { startOffset: range.startOffset, endOffset: range.endOffset, newText } },
    window.location.origin
  );
}
```

> 约定：发送端 target origin 一律用 `window.location.origin`（不用 `'*'`）；接收端一律校验 `source===window`、`origin===location.origin`、自定义 `source` 标识（`KFE_INJECTED`/`KFE_CONTENT`）。injected.js 端已做对称校验（见 §3.2）。

#### 3.3.2 点击位置 → 字段定位（核心算法）

```js
import { parseTree, findNodeAtOffset, getNodePath } from 'jsonc-parser';

function locateField(fullText, offset) {
  const root = parseTree(fullText);             // 解析 AST
  const node = findNodeAtOffset(root, offset);  // 点击命中的节点
  if (!node) return null;

  const path = getNodePath(node);  // 如 ['hits','hits',0,'_source','user','name']
  return analyze(node, path, root);
}

function analyze(node, path, root) {
  // 1. 校验路径形如 hits.hits[N]._source.xxx
  const i = path.indexOf('_source');
  if (i < 1 || path[i-2] !== 'hits' || path[i-1] === undefined) return null;
  // path 结构: [..., 'hits', 'hits', N, '_source', ...field...]
  // 校验 hits.hits 数组
  // ...

  // 2. 取得该 hit 的 _id / _index（从同级节点读取）
  const hitNode = getHitNode(root, path);   // 定位到 hits.hits[N] 对象
  const _id = readProp(hitNode, '_id');
  const _index = readProp(hitNode, '_index');

  // 3. 判断点击的是 key 还是 value, 是标量还是数组元素
  const targetField = resolveEditTarget(node, path, i);
  if (!targetField) return null;  // 命中 key 名 / 不可编辑 → 不触发

  return {
    _id, _index,
    fieldPath: targetField.dotPath,        // 'user.name' 或 'tags'
    valueType: targetField.type,           // 'scalar' | 'array'
    originalValue: targetField.rawValue,   // 原值(JS 值)
    valueRange: targetField.offsetRange    // { startOffset, endOffset } 用于替换
  };
}
```

**`resolveEditTarget` 规则**：
- 命中节点是 **property 的 value**（标量）→ 返回该标量，`type='scalar'`，range = 标量 value 的 offset 范围。
- 命中节点是 **数组内的元素**（标量或对象）→ 向上找到数组节点，返回整个数组，`type='array'`，range = 整个数组 `[...]` 的 offset 范围。
- 命中节点是 **property 的 key**（字段名）→ 取其 value：
  - value 是标量 → 同标量处理。
  - value 是数组 → **不触发**（MVP）。
  - value 是对象 → **不触发**（MVP）。
- 命中元数据字段（`_id`/`_index`/`_score`/`_type`，即不在 `_source` 子树内）→ 不触发。

#### 3.3.4 弹窗与提交
```js
async function submitUpdate(ctx, inputText) {
  // 0. 前置校验: _index / _id 必须存在, 否则无法定位文档
  if (!ctx._index || !ctx._id) {
    showErrorInModal('无法定位文档：缺少 _index 或 _id（该 hit 可能不包含元数据）');
    return;
  }

  // 1. 类型转换
  const newValue = convertByType(ctx.originalValue, inputText, ctx.valueType);
  // 抛错则在弹窗内提示, return

  // 2. 组装 body
  const body = { doc: setByPath({}, ctx.fieldPath, newValue) };
  // 注: ES _update 支持点号 key, 也可直接 { doc: { 'user.name': v } }

  // 3. 调用 Kibana proxy
  const resp = await callProxy(
    `/${encodeURIComponent(ctx._index)}/_update/${encodeURIComponent(ctx._id)}`,
    'POST', body
  );

  // 4. 处理结果
  if (resp.ok) {
    showResultInModal(resp.json);            // 展示成功响应
    // 经 postMessage(REPLACE) 通知 injected.js 调 session.replace 替换 Ace 文本
    postReplaceToAce(ctx.valueRange, serialize(newValue, ctx.valueType));
  } else {
    showResultInModal(resp.json, true);      // 仅展示错误, 不动 Ace 编辑器
  }
}
```

### 3.4 类型转换 convertByType

```js
function convertByType(originalValue, input, valueType) {
  if (valueType === 'array') {
    try { return JSON.parse(input); }
    catch { throw new Error('JSON 格式错误'); }
  }
  const t = originalValue === null ? 'null' : typeof originalValue;
  switch (t) {
    case 'number': {
      const n = Number(input);
      if (input.trim() === '' || Number.isNaN(n)) throw new Error('请输入合法数字');
      return n;
    }
    case 'boolean': {
      const v = input.trim().toLowerCase();
      if (v === 'true') return true;
      if (v === 'false') return false;
      throw new Error('请输入 true 或 false');
    }
    case 'string':
      return input;
    case 'null':
      // 智能推断
      const v = input.trim();
      if (v === 'null') return null;
      if (v.toLowerCase() === 'true') return true;
      if (v.toLowerCase() === 'false') return false;
      if (v !== '' && !Number.isNaN(Number(v))) return Number(v);
      return input;
  }
}
```

### 3.5 Kibana Proxy 调用

```js
async function callProxy(path, method, body) {
  const url = `/api/console/proxy?path=${encodeURIComponent(path)}&method=${method}`;
  const res = await fetch(url, {
    method: 'POST',                     // proxy 统一用 POST
    headers: {
      'Content-Type': 'application/json',
      'kbn-xsrf': 'true'                // 必须, 否则 400
    },
    credentials: 'include',             // 复用 session cookie
    body: JSON.stringify(body)
  });
  const json = await res.json().catch(() => ({}));
  return { ok: res.ok && !json.error, json };
}
```

> **版本探测**（FR-02）：激活后调用 `callProxy('/', 'GET')`，从返回的 `version.number` 解析 ES 版本，仅用于日志和潜在兼容分支。

#### 3.5.1 两种更新模式（doc vs painless script）

locate 算法根据字段路径是否含**数组下标**，返回 `updateMode`：

```js
// 模式一：doc 部分更新（字段不在任何数组内）
// 适用：顶层标量、嵌套对象内标量、顶层数组整体编辑
function updateField(index, id, dotPath, value) {
  return callProxy(`/${encodeURIComponent(index)}/_update/${encodeURIComponent(id)}`,
    'POST', { doc: { [dotPath]: value } });
}

// 模式二：painless script 更新（字段在数组内，路径含下标）
// 适用：数组内对象的标量字段，如 WorkExperienceList[0].StartDate
function updateByScript(index, id, painlessPath, value) {
  return callProxy(`/${encodeURIComponent(index)}/_update/${encodeURIComponent(id)}`,
    'POST', {
      script: {
        lang: 'painless',
        source: `${painlessPath} = params.v`,   // 如 ctx._source['list'][0]['field'] = params.v
        params: { v: value }                    // 通过 params 传值，类型天然正确、无需转义
      }
    });
}
```

> **为什么数组内字段必须用 script？**
> ES `_update` 的 `doc` 合并是基于字段名的对象合并，**无法表达"数组第 N 个元素的某字段"**。
> 若对数组用点号路径 `{ "doc": { "list.field": v } }`，ES 找不到该字段，合并后无变化 → 返回 `result:"noop"`（实际未修改）。
> 因此数组内字段改用 painless：`ctx._source['list'][0]['field'] = params.v`，按下标精准赋值。
>
> **painless 路径生成**：key 一律用 `['key']` 方括号字面量访问（兼容含点/空格的字段名），下标用 `[N]`；
> 值通过 `params.v` 传递，避免在 source 字符串里手工拼接和转义（数字/布尔/字符串/null/数组类型均由 JSON 自动携带）。

## 5. 兼容性与风险

| 风险 | 等级 | 应对 |
|------|------|------|
| 拿不到 Ace 实例 | **低（已实测可用）** | `window.ace` 全局存在，`ace.edit('.conApp__output')` 可反查结果区实例；若失败退化为监听 DOM `mousedown` + `caretRangeFromPoint` |
| 结果区与请求区区分 | **低（已实测）** | 直接用 DOM class：结果区 `.conApp__output`，请求区 `#ConAppEditor`；忽略 `.ace_autocomplete` |
| 结果区只读导致无法替换文本 | 低 | 替换前 `setReadOnly(false)`，替换后立即恢复 |
| SPA 路由：从其他子页切到 Console | 中 | content script 监听 `hashchange`，hash 含 `dev_tools/console` 时（重新）初始化；离开时清理浮层/监听 |
| Ace 虚拟渲染导致 DOM 行不全 | 低 | 已规避：始终用 `session.getValue()` 全文解析，不依赖可见 DOM 行 |
| 严格 CSP 拦截注入脚本 | 低 | 使用 `web_accessible_resources` + 外部 js 文件注入（非 inline） |
| `_index` 为别名/数据流导致 `_update` 失败 | 低 | 使用 hit 自身 `_index`（指向具体索引），异常由 ES 报错兜底 |
| 未来 Kibana 升级到 Monaco（≥7.11） | 中 | 编辑器交互层已抽象为 `injected.js` 适配层；升级时只需替换该层（getEditors/onMouseDown/executeEdits），上层 AST/UI/proxy 逻辑不变 |
